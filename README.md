# 在线聊天室 · 使用说明

一个可以直接在你自己电脑上运行的实时聊天服务器，基于 Node.js + WebSocket。
支持：多人实时聊天、昵称、在线用户列表、"正在输入"提示、消息历史（重启后清空）、**PWA（可添加到手机主屏幕，像 App 一样打开）**、**@提及提醒**、**置顶公告**。

## 新增：@提及与置顶公告

- **@提及别人**：在输入框打 `@` 会自动弹出在线用户列表，用方向键选或直接点选，插入 `@用户名`。被@的人消息气泡会有橙色高亮边框，**并有两次脉冲闪光动画 + "叮咚"提示音**（声音第一次需要先加入聊天室触发过一次点击，才能绕开浏览器的自动播放限制，正常操作流程下不用额外处理）；如果对方当时没在看这个标签页，浏览器标签标题还会闪烁提示"💬 有人@你了"，切回来自动恢复。
- **置顶公告**：聊天窗口顶部有一条常驻公告栏，任何人都能点"编辑"设置/修改，保存后所有人实时同步看到，适合发通知、规则、置顶消息等。
- **常用文本**：聊天窗口最右侧一栏，用来存放自己常用的短语（比如"收到"、"稍等"），点一下会把文字填进输入框（不会直接发送，还能改），可以自己增删。**这个是保存在浏览器本地的，每个人看到的、能用的常用语只有自己设置的那些，不会跟别人共享。**
- **引用回复**：每条消息下面有个"↩ 回复"按钮，点一下会在输入框上方出现一条引用预览（原消息的人+内容），此时发送的新消息会带着这段引用一起显示给所有人，方便在多人讨论时明确"回的是哪条"。点预览条右边的"✕"可以取消引用。

## 〇、别人如何"像用 App 一样"加入

1. 用手机浏览器打开聊天室地址（局域网地址或内网穿透给的公网地址）
2. **iPhone（Safari）**：点底部分享图标 → "添加到主屏幕"
   **安卓（Chrome）**：点右上角菜单 ⋮ → "添加到主屏幕" / "安装应用"
3. 桌面会出现一个聊天室图标，以后点它直接全屏打开，没有浏览器地址栏，跟原生 App 体验一致
4. 首次打开时输入昵称即可加入聊天

## 一、本地运行

1. 确保电脑已安装 [Node.js](https://nodejs.org/)（建议 18+ 版本）
2. 打开终端，进入项目文件夹：
   ```bash
   cd chat-app
   npm install
   npm start
   ```
3. 启动后终端会显示：
   ```
   本机访问: http://localhost:3000
   局域网访问: http://<你的局域网IP>:3000
   ```
4. 浏览器打开 `http://localhost:3000` 即可使用。

如果 3000 端口被占用，可以指定其他端口：

```bash
PORT=8080 npm start
```
（Windows 用 `set PORT=8080 && npm start`）

## 二、让局域网内其他人访问（同一 WiFi/路由器下）

1. 查看你电脑的局域网 IP：
   - Windows：`ipconfig` 看"IPv4 地址"
   - Mac/Linux：`ifconfig` 或 `ip addr` 看 `192.168.x.x` 或 `10.x.x.x`
2. 让防火墙放行该端口（Windows 首次启动 Node 时会弹出防火墙提示，允许即可）
3. 其他人在浏览器输入 `http://你的局域网IP:3000` 即可加入聊天

## 三、让公网（外网）用户也能访问

由于服务器在你自己电脑上，外网用户无法直接访问局域网 IP，需要以下任一方式"暴露"到公网：

### 方式 A：内网穿透工具（最简单，推荐新手）

无需路由器设置，几分钟搞定，缺点是免费版通常有速度限制或临时域名。常见工具：

- **ngrok**：`ngrok http 3000`，会给你一个 `https://xxxx.ngrok-free.app` 的公网地址
- **frp**（花裤衩开发，国内常用，可自建服务器获得固定域名）
- **cpolar / natapp**：国内内网穿透工具，中文文档友好

### 方式 B：路由器端口转发（DDNS）

1. 登录路由器管理后台，设置"端口转发/虚拟服务器"，将公网端口（如 3000）转发到你电脑的局域网 IP:3000
2. 如果你的公网 IP 会变化，配合花生壳、DDNS-GO 等动态域名服务绑定一个固定域名
3. 注意：需要你的宽带有公网 IP（部分家庭宽带是运营商 NAT，没有公网 IP，此方式不可用）

### 方式 C：云服务器中转（更稳定，适合长期使用）

如果需要 7x24 稳定访问，建议最终把 `server.js` 和 `public/` 部署到一台云服务器（阿里云/腾讯云等），
而不是长期依赖家用电脑 + 内网穿透。代码无需修改，直接上传后 `npm install && npm start` 即可。

## 四、安全提示

- 当前版本**没有身份验证**，任何拿到链接的人都能加入并看到聊天记录，请勿用于敏感信息交流
- 若长期暴露在公网，建议后续加上：登录密码 / HTTPS（可配合 Caddy、Nginx 反向代理自动签发证书）/ 限流防刷
